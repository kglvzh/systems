## Обнаружение отказов в распределенных системах (протокол Gossip)

## Цель работы
Изучить принципы обнаружения отказов в распределенных системах на примере симуляции протокола Gossip (Serf), сравнить с Heartbeat и Ping, исследовать влияние параметров на конвергенцию и нагрузку.

## Индивидуальное задание (вариант №3)
### Формулировка
Оптимизация Gossip Fanout: исследовать влияние Fanout (2..10) на время конвергенции и трафик.

### Параметры эксперимента
| Параметр | Значение |
|----------|----------|
| Gossip Interval | 0.2 сек |
| Gossip Fanout | 2,3,4,5,6,7,8,9,10 |
| Количество узлов (N) | 100 |
| Отказавшие узлы | 5% (5 узлов) |
| Потеря пакетов | 0% |

## Теоретический расчет полосы пропускания
Формула: Bandwidth (бит/с) = (ActiveNodes × Fanout × PacketSize × Overhead × 8) / Interval  
ActiveNodes = 95, PacketSize = 1024 байт, Overhead = 1.2

| Fanout | Bandwidth (бит/с) |
|--------|------------------|
| 2 | 933 888 |
| 4 | 1 867 776 |
| 6 | 2 801 664 |
| 8 | 3 735 552 |
| 10 | 4 669 440 |

## Реализация протоколов
- **Gossip (Serf)**: каждый узел обменивается с fanout случайными соседями, информация распространяется вирусно.
- **Heartbeat (Full-mesh)**: каждый узел пингует всех остальных каждый такт, трафик O(N²).
- **Ping (Random)**: каждый узел пингует одного случайного соседа, трафик O(N).

## Вывод
При нулевых потерях пакетов все три протокола обнаруживают отказ за один раунд, поэтому решающим фактором становится трафик: Heartbeat неприменим для больших сетей из-за квадратичного роста O(N²), Ping экономичен но слишком медленен в реальных условиях с потерями, а Gossip обеспечивает наилучший баланс; при оптимизации Fanout для сети из 100 узлов с 5% отказов увеличение Fanout выше 2–3 не ускоряет конвергенцию, но линейно повышает трафик, поэтому минимальный Fanout является оптимальным при отсутствии потерь пакетов.
