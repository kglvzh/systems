## Тема: gRPC-сервис InventoryControl (Client Streaming RPC)

---

## Задание
Реализовать gRPC-сервис InventoryControl с методом:

**BulkUpdateStock(stream StockItem)**
Метод должен принимать поток товаров от клиента и массово обновлять остатки на складе (Client streaming RPC).

Дополнительно реализован метод:
- **ClearData** — для очистки всех данных на сервере
- **Сохранение данных в файл** — на стороне клиента

---

## Архитектура решения
В работе реализована классическая клиент-серверная архитектура.

```
+-------------+        gRPC         +-------------+
|   Клиент    |  <--------------->  |   Сервер    |
|  client.py  |    Client stream    |  server.py  |
+-------------+                     +-------------+
       |                                  |
       |                                  |
+-------------+                     +-------------+
|  items.dat  |                     |  stock_db   |
| (файл)      |                     | (в памяти)  |
+-------------+                     +-------------+
```

### Компоненты системы
- **Client (client.py)** — отправляет поток товаров на сервер, сохраняет данные локально в файл.
- **Server (server.py)** — принимает поток товаров, обрабатывает и хранит их в памяти.
- **inventory.proto** — контракт взаимодействия, описывает методы и структуры.
- **items.dat** — файл для хранения товаров на стороне клиента между запусками.
- **stock_db** — хранилище данных на сервере (в оперативной памяти).

---

## Описание реализации
В рамках лабораторной работы был создан gRPC-сервис InventoryControl.

### Описание сервиса выполнено в файле:
**inventory.proto**

Сервис содержит два метода:
- **BulkUpdateStock** — принимает поток товаров (Client streaming RPC);
- **ClearData** — очищает все данные на сервере.

Метод BulkUpdateStock использует ключевое слово **stream** для входящих сообщений, что позволяет клиенту отправлять несколько товаров в рамках одного соединения.

---

## Генерация gRPC-кода
После создания inventory.proto была выполнена команда:

```bash
python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. inventory.proto
```

В результате были автоматически созданы файлы:
- **inventory_pb2.py** — классы сообщений
- **inventory_pb2_grpc.py** — классы сервиса

Эти файлы обеспечивают взаимодействие клиента и сервера через gRPC.

## Реализация сервера
Серверная часть реализована в файле:

**server.py**

Создан класс **InventoryControlServicer**, наследующийся от InventoryControlServicer.

Реализованы методы:
- **BulkUpdateStock** — принимает поток товаров, валидирует данные и сохраняет в `stock_db`;
- **ClearData** — очищает хранилище данных на сервере.

Данные хранятся в словаре `self.stock_db`, который сохраняется между вызовами.

Сервер запускается на порту **50051** и ожидает подключения клиентов.
